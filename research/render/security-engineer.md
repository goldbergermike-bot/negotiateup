#### Negotiation DNA — Why This Role Commands a Premium at Render

Render is a private startup headquartered in San Francisco building a modern cloud platform designed as the successor to Heroku. The platform provides web services, static sites, background workers, cron jobs, managed PostgreSQL, Redis, and private networking — all deployed directly from Git with zero-configuration.

Security Engineers at Render own the security posture of a multi-tenant cloud platform. This is among the most critical roles at any infrastructure company — a security breach doesn't just impact Render, it impacts every application and database running on the platform. The stakes are existential: customer trust in a cloud platform depends entirely on security.

The security scope at Render is broad and deep: container isolation and escape prevention, network segmentation between tenants, build pipeline security (supply chain attacks), managed database encryption and access control, secrets management, SOC 2 compliance, and incident response. At a startup, one security engineer covers what would be an entire security organization at a larger company.

Security engineering talent is scarce, and cloud infrastructure security talent is even scarcer. Render competes with Big Tech security teams (which offer higher base salaries) and security-focused companies. The startup equity upside and platform-wide ownership create Render's value proposition.

#### Lever 1: Multi-Tenant Platform Security

"Render runs thousands of customer applications on shared infrastructure. I design and maintain the container isolation, network segmentation, and access controls that ensure strict tenant separation. A security failure here is existential for the company."

#### Lever 2: Build Pipeline & Supply Chain Security

"Render's git-push-to-deploy model means customer code flows through our build infrastructure. I secure the entire build pipeline — from git webhook to running container — preventing supply chain attacks that could compromise customer applications."

#### Lever 3: Compliance & Customer Trust

"Enterprise customers require SOC 2, and increasingly SOC 2 Type II, before trusting a cloud platform with production workloads. I build and maintain the compliance frameworks that unlock enterprise revenue."

#### Lever 4: Scarce Talent & Startup Ownership

"Cloud infrastructure security engineers are among the hardest roles to hire. I bring platform security expertise that directly protects Render's reputation and revenue. I expect equity reflecting both talent scarcity and the existential nature of this work."
